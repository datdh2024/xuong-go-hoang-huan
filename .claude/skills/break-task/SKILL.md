----
name: break-task
description: Use this skill when you need to break a task into smaller steps.
---

- separate the task into smaller user-stories by vision of user (or other stakeholders)
- Ask user to confirm the user-stories and update by user comment
- Arrange user-stories by priority (high, medium, low) and risk (high, medium, low). high priority and high risk first
- Sort by Agile method, each story should bring value to user, testable, don't include technical information
- save to docs/user-stories/US-XX-\*.md
