----
name: break-task
description: Use this skill when you need to break a parent Epic/Story/Task into smaller steps.
---

- You get a Jira Epic/Story from user, then separate the task into smaller user-stories by vision of user (or other stakeholders)
- Find related user flow in docs/user-flows/, then add link to user-stories
- Ask user to confirm the user-stories and update by user comment
- Arrange user-stories by priority (high, medium, low) and affect (high, medium, low). high priority and high affect first
- Apply Agile method, each story should bring value to user, testable, don't include technical information
- save to Jira with parent Epic/Story as Story/Task
